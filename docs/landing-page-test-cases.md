| **Test Case No.** | **Module Name**     | **Test Case Description**                                                                                                           | **Acceptance Criteria**                                                                                                       |
| ----------------- | ------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **T_C_1.1**       | Initial Access      | Verify that accessing the site via the correct production URL (e.g., `http://www.example.com`) successfully loads the Landing Page. | The correct Landing Page content must load without errors.                                                                    |
| **T_C_1.2**       | Protocol            | Verify that accessing the site via HTTP (`http://example.com`) automatically redirects to the secure HTTPS protocol.                | The browser must show `https://example.com` with a secure connection indicator (e.g., lock icon).                             |
| **T_C_1.3**       | URL Redirection     | Verify that accessing the site via a non-preferred domain format redirects to the canonical URL.                                    | Accessing `example.com` must redirect to `www.example.com` (or vice-versa), based on the system standard.                     |
| **T_C_1.4**       | Overall Page        | Verify the landing page loads successfully and quickly on all target devices/browsers.                                              | The page must load within a reasonable time (e.g., < 3 seconds), and no broken elements or console errors should be observed. |
| **T_C_1.5**       | Navbar              | Verify the Home Tab/Logo link in the Navbar redirects the user back to the landing page.                                            | Clicking the Home link/Logo should reload or remain on the landing page URL.                                                  |
| **T_C_1.6**       | Navbar              | Verify the Login button in the Navbar correctly redirects the user to the dedicated Login page.                                     | Clicking the Navbar Login button must redirect to the correct Login URL (e.g., `/login`).                                     |
| **T_C_1.7**       | Navbar              | Verify the Navbar remains sticky/fixed when the user scrolls down the page (if designed to be).                                     | The Navbar should stay visible at the top of the viewport as the user scrolls.                                                |
| **T_C_1.8**       | Hero Section        | Verify the main heading and text in the Hero section are displayed correctly and are contextually accurate.                         | The heading and subtext must be legible, use intended fonts, and correctly represent the site's purpose.                      |
| **T_C_1.9**       | Hero Section        | Verify the Login button in the Hero section correctly redirects the user to the dedicated Login page.                               | Clicking the Hero Login button must redirect to the correct Login URL (e.g., `/login`).                                       |
| **T_C_1.10**      | Description Section | Verify all content (text, images, or media) in the Description section is fully displayed and properly formatted.                   | All content must be visible, properly aligned, and images/videos must load without error.                                     |
| **T_C_1.11**      | FAQ Section         | Verify that each question in the FAQ section expands/collapses correctly (accordion behavior).                                      | Clicking a question must reveal its answer; clicking again or another question must collapse previously open content.         |
| **T_C_1.12**      | Footer              | Verify all footer text/copyright information is displayed correctly.                                                                | The footer must show the current year and correct organization name.                                                          |
| **T_C_1.13**      | Footer              | Verify all internal footer links (Privacy Policy, Terms of Service, etc.) are functional.                                           | Clicking each internal link must successfully load the respective page or document.                                           |
